**Last Updated:** 2025-12-06

## Date & Time Formats

### Standard Formats:
- **Date:** `dd/mm/yyyy` (e.g., 06/12/2025)
- **Time:** `hh:mm` (24-hour format, e.g., 14:30)
- **DateTime:** `dd/mm/yyyy hh:mm` (e.g., 06/12/2025 14:30)

### Examples:
```json
{
  "check_in": "10/12/2025",
  "check_out": "12/12/2025",
  "departure_time": "14:30",
  "created_at": "06/12/2025 18:03"
}
```

### Database Storage:
- Database stores as PostgreSQL `DATE` or `TIMESTAMP`
- API converts to dd/mm/yyyy format in responses
- API accepts dd/mm/yyyy format in requests

### Implementation:
```javascript
// Format date for API response
function formatDate(date) {
  const d = new Date(date);
  const day = String(d.getDate()).padStart(2, '0');
  const month = String(d.getMonth() + 1).padStart(2, '0');
  const year = d.getFullYear();
  return `${day}/${month}/${year}`;
}

// Format time for API response
function formatTime(date) {
  const d = new Date(date);
  const hours = String(d.getHours()).padStart(2, '0');
  const minutes = String(d.getMinutes()).padStart(2, '0');
  return `${hours}:${minutes}`;
}

// Format datetime for API response
function formatDateTime(date) {
  return `${formatDate(date)} ${formatTime(date)}`;
}

// Parse date from dd/mm/yyyy to database format
function parseDate(dateString) {
  // Input: "10/12/2025"
  const [day, month, year] = dateString.split('/');
  return `${year}-${month}-${day}`; // Output: "2025-12-10"
}
```

## Currency Format

### Standard Format:
- **Display:** `$1,234.56` (with comma separators)
- **Database:** DECIMAL(12,2) (stores as 1234.56)

### Examples:
```json
{
  "total_sell_price": "2,500.00",
  "total_cost_price": "1,800.00",
  "gross_profit": "700.00"
}
```

## Booking Code Format

### Standard Format:
- **Pattern:** `Funny-XXXX`
- **Starting Number:** 1046
- **Examples:** Funny-1046, Funny-1047, Funny-1048

### Auto-generated by Database:
```sql
SELECT generate_booking_code(); -- Returns: Funny-1046
```

## Status Values

### Booking Status:
- `inquiry` - Initial request
- `quoted` - Quotation sent
- `confirmed` - Client accepted
- `completed` - Tour finished
- `cancelled` - Booking cancelled

### Payment Status:
- `pending` - No payment received

### Resource Status:
- `available` - Ready to use
- `in_use` - Currently assigned

## Validation Rules

### Date Validation:
- Format: dd/mm/yyyy
- Valid date ranges
- Check-out must be after check-in
- Travel dates in the future

### Time Validation:
- Format: hh:mm (24-hour)
- Range: 00:00 to 23:59

### Amount Validation:
- Must be positive number
- Maximum 2 decimal places
- Range: 0.01 to 999,999.99
